## Rationale

### Missing `dcs hunt-fuzz` in README Quick Start
The CLI command exists (cli.py:674) and is documented in CLAUDE.md (line 92), but is missing from the user-facing README Quick Start examples. This is the primary entry point for users wanting to try the integrated workflow.

### Missing `deep_scan_hunt_fuzz` MCP tool in README
The MCP server conditionally registers this tool when ContainerBackend and Anthropic SDK are available (server.py:374-431). The README lists 7 MCP tools but only documents 7 tools without including `deep_scan_hunt_fuzz`. This creates a documentation gap for MCP users.

### Missing `DCS_BRIDGE_MAX_TARGETS` environment variable in README
This variable is documented in CLAUDE.md (line 127) and is used in the CLI (cli.py:687-688) and MCP tool (server.py:418-422). It should be documented in the README's environment variables section for consistency.

### Architecture diagram suggestion
The bridge module represents a significant architectural addition (5 new files: models.py, resolver.py, orchestrator.py, cwe_guidance.py, __init__.py). While the module is listed in CLAUDE.md's architecture tree (line 24), the README could benefit from a visual workflow diagram showing how HUNTER → BRIDGE → FUZZER → CORRELATE flows, especially since this is a distinct operational mode from running SAST or fuzzing independently.
